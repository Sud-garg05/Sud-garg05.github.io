Core AI / LLM layer
LLM
Claude (Anthropic) or GPT-4o
CV parsing, scoring, JD gen, copilot chat — strong at structured extraction
Embeddings
text-embedding-3-large or Cohere embed-v3
Semantic search over resumes, JDs, candidate history
Orchestration
LangChain or LlamaIndex
RAG pipelines, tool chaining, memory across recruiter sessions
Vector DB
Pinecone or pgvector (Postgres)
Resume + JD semantic retrieval. pgvector if already on Postgres
Prompt mgmt
Langfuse or PromptLayer
Prompt versioning, A/B testing, output drift monitoring
Backend & API
API
Python — FastAPI

Top risks and mitigations
RiskSeverityMitigation
Bias in CV screening — model discriminates by gender, name, university
High
Anonymise resumes before LLM call; mandatory bias audit before beta; human-in-the-loop for final shortlist
GDPR / data privacy — candidate PII sent to third-party LLM API
High
Sign DPA with Anthropic/OpenAI; anonymise before API call; use EU data residency endpoints; DPO sign-off
LLM hallucination on scorecards — fabricated candidate attributes
High
JSON schema validation on all outputs; confidence scoring; human review for any flagged outputs
ATS integration failure — Workday / Greenhouse API changes break pipeline
Medium
Versioned API contracts; webhook fallback; integration tests in CI/CD
Stakeholder resistance from recruiters — fear of job replacement
Medium
Frame as copilot not replacement; involve recruiters in UAT design; show time saved per week
API cost overrun — more CVs than estimated at launch
Medium
Per-role token budgets; caching for repeat queries; cost alerts at 80% of monthly budget
Prompt drift — LLM behaviour changes after model update
Medium
Langfuse prompt versioning; regression test suite on golden eval set; pin model versions
Scope creep in alpha — stakeholders add features mid-build
Low
Strict change control after Phase 3 approval; new requests go to backlog v2
Legal & compliance checklist
GDPR Article 22 — automated decision-making
AI screening must be advisory only; candidates must be able to request human review
EU AI Act — high-risk system classification
Recruitment AI may be classified high-risk; requires transparency, logging, human oversight
Data Processing Agreement with LLM vendor
Anthropic / OpenAI DPA must be signed before any candidate data hits the API
Candidate transparency notice
Privacy policy must disclose that AI is used in screening
Audit trail retention
All AI decisions logged with inputs/outputs for minimum 12 months
