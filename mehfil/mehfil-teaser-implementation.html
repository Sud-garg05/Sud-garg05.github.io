Mehfil Waitlist
One-page hero, Spotify-powered, WhatsApp-viral

A single hero section that turns a music-app login into a shareable "song for the Mehfil" card and a skip-the-line referral loop — engineered to be dropped as a party poll into WhatsApp groups.

01Concept

No email form. The signup is the music connect. One tap reads the user's vibe, brands them a "song for the Mehfil," gives them a waitlist position, and arms a WhatsApp poll that drops their friends straight into the same line — jumping them up it.

🎯 The hook

"This is your song for the Mehfil" — a personalized, branded music card people want to post.

🔥 The FOMO

Live city counter + a high waitlist number that stings → motivates the share.

🪜 The engine

Skip-the-line: every friend who joins via your link jumps you up the queue.

💬 The loop

Share = a WhatsApp party poll. "Who's IN?" — the group sees who's out.

Land. Hero loads with a live counter — "4,212 already in line in your city. Your friends are inside." No form. Three buttons: Connect Spotify · YouTube Music · Email.
Connect = sign up. One OAuth tap is identity and vibe data. Loader: "Reading your vibe…"
Dashboard reveal. Their Mehfil card: archetype + genres + "🔥 This is your song for the Mehfil" with the track playing in an embedded Spotify player.
Position sting. "You're #4,213." Far back on purpose — that tension drives step 5.
Drop the party poll. CTA opens WhatsApp pre-filled with a poll-style message that includes their song + referral link.

03Architecture

Static front-end + one tiny backend for the only two things that truly need a server: the referral ledger and the OAuth token exchange.

Front-end

Plain HTML/CSS/JS or Next.js. Spotify iframe embed for playback — zero audio hosting.

Backend

Vercel/Cloudflare function + Supabase (Postgres) or Upstash KV. ~3 endpoints.

Why a backend at all

Referral counts must be server-authoritative (can't trust the client), and the Spotify client secret can't ship to the browser.

v1 shortcut

Skip the backend entirely — fake the counter + position in JS to ship the teaser today.

05Auth — Spotify OAuth (PKCE)

PKCE is the right flow for a public client. Step 1 redirects to Spotify; step 2 swaps the code for a token (secret stays server-side).

auth.js — kick off PKCE loginconst CLIENT_ID = "YOUR_SPOTIFY_CLIENT_ID";
const REDIRECT  = location.origin + "/callback";
const SCOPES    = "user-top-read user-read-recently-played";

function base64url(buf){
  return btoa(String.fromCharCode(...new Uint8Array(buf)))
    .replace(/\+/g,"-").replace(/\//g,"_").replace(/=+$/,"");
}
async function loginWithSpotify(){
  // 1. create + stash the PKCE verifier
  const verifier = base64url(crypto.getRandomValues(new Uint8Array(64)));
  const challenge = base64url(await crypto.subtle.digest(
    "SHA-256", new TextEncoder().encode(verifier)));
  sessionStorage.setItem("pkce_verifier", verifier);

  // 2. bounce the user to Spotify's consent screen
  const url = new URL("https://accounts.spotify.com/authorize");
  url.search = new URLSearchParams({
    client_id: CLIENT_ID, response_type: "code", redirect_uri: REDIRECT,
    scope: SCOPES, code_challenge_method: "S256", code_challenge: challenge,
  });
  location.href = url.toString();
}
api/auth/spotify.js — token swap (serverless)// POST { code, verifier } -> { access_token }
export default async function handler(req, res){
  const { code, verifier } = req.body;
  const body = new URLSearchParams({
    grant_type: "authorization_code",
    code, redirect_uri: process.env.REDIRECT_URI,
    client_id: process.env.SPOTIFY_CLIENT_ID,
    code_verifier: verifier,
  });
  const r = await fetch("https://accounts.spotify.com/api/token", {
    method: "POST",
    headers: { "Content-Type": "application/x-www-form-urlencoded" },
    body,
  });
  const tok = await r.json();          // { access_token, expires_in, ... }
  res.json({ access_token: tok.access_token });
}
vibe.js — read top track + genresasync function readVibe(token){
  const h = { headers: { Authorization: `Bearer ${token}` } };
  const [tracks, artists] = await Promise.all([
    fetch("https://api.spotify.com/v1/me/top/tracks?limit=5&time_range=short_term", h).then(r=>r.json()),
    fetch("https://api.spotify.com/v1/me/top/artists?limit=10&time_range=short_term", h).then(r=>r.json()),
  ]);
  const topTrack = tracks.items[0];                  // the "song for the Mehfil"
  const genres = [...new Set(artists.items.flatMap(a => a.genres))];
  return {
    trackId:  topTrack.id,
    trackName: topTrack.name,
    artist:   topTrack.artists[0].name,
    genres,
    archetype: pickArchetype(genres),                // see §6
  };
}

export function pickArchetype(genres = []){
  const g = genres.map(x => x.toLowerCase());
  for (const a of ARCHETYPES)
    if (a.match.some(m => g.some(x => x.includes(m)))) return a;
  return ARCHETYPES[Math.floor(Math.random() * ARCHETYPES.length)]; // playful fallback
}

Autoplay reality: browsers block audio until a user gesture. Since the user clicked "Connect" to reach the dashboard, that gesture usually counts — but always keep the embed's own play button visible as the guaranteed path.

08WhatsApp party-poll share

WhatsApp can't pre-fill its native poll widget via a link — only text. So we pre-fill a poll-styled message through wa.me that includes the user's song, archetype, position, and referral link.

share.jsfunction buildPollMessage({ song, artist, archetype, position, refLink }){
  return (
`🍻 *MEHFIL — party poll. Who's IN?*

I just claimed my spot (${position}) 🎵
My song for the Mehfil: *${song}* — ${artist}
My vibe: ${archetype}

Vote 👇 + grab yours:
✅ I'm IN → ${refLink}
😴 I'm out (ok loser)

Last one to join buys the first round 👀`
  );
}

function shareToWhatsApp(vibe, pos, refCode){
  const refLink = `https://mehfil.com/?ref=${refCode}`;
  const text = buildPollMessage({
    song: vibe.trackName, artist: vibe.artist,
    archetype: vibe.archetype.name, position: pos, refLink,
  });
  // opens WhatsApp (mobile app / web) with the message pre-typed
  location.href = `https://wa.me/?text=${encodeURIComponent(text)}`;
}

Optional "real poll" path: render a Copy poll options button (I'm in / Maybe / I'm out) and tell the user to paste them into a native WhatsApp group poll — more authentic, one extra step. Link-unfurl preview (the song image) is set via Open Graph meta tags on the landing page.

09Referral waitlist — the line-jump

Position is server-authoritative. Joining via someone's ?ref= credits the referrer and moves them up a fixed number of spots.

api/waitlist/join.jsconst JUMP_PER_REFERRAL = 300;

// POST { archetype, trackId, ref }  ->  { position, refCode }
export default async function join(req, res){
  const { archetype, trackId, ref } = req.body;
  const refCode = crypto.randomUUID().slice(0, 8);

  // 1. create the row at the back of the line
  const { id, joined_at } = await db.insert("waitlist", {
    ref_code: refCode, referred_by: ref ?? null, archetype, track_id: trackId,
  });

  // 2. if they came from a referral, move the referrer up
  if (ref) await db.exec(
    `UPDATE waitlist SET boost = boost + $1 WHERE ref_code = $2`,
    [JUMP_PER_REFERRAL, ref]);

  const position = await positionFor(refCode);
  res.json({ position, refCode });
}

// effective position = signup order minus the spots you've earned
async function positionFor(refCode){
  const row = await db.one(
    `SELECT joined_at, boost FROM waitlist WHERE ref_code = $1`, [refCode]);
  const ahead = await db.count(
    `SELECT 1 FROM waitlist WHERE joined_at < $1`, [row.joined_at]);
  return Math.max(1, ahead + 1 - row.boost);
}
live-position.js — front-end poll + animated barasync function refreshPosition(refCode){
  const { position } = await fetch(`/api/waitlist/position?ref=${refCode}`).then(r=>r.json());
  const el = document.getElementById("pos");
  const from = parseInt(el.dataset.v || position, 10);
  animateCount(el, from, position);          // satisfying climb when friends join
  el.dataset.v = position;
}
setInterval(() => refreshPosition(localStorage.ref), 8000);

function animateCount(el, from, to){
  const t0 = performance.now(), dur = 600;
  (function tick(t){
    const k = Math.min(1, (t - t0) / dur);
    const v = Math.round(from + (to - from) * k);
    el.textContent = "#" + v.toLocaleString();
    if (k < 1) requestAnimationFrame(tick);
  })(performance.now());
}
Anti-abuse: credit a referral only on a real signup (validated music-connect or verified email), de-dupe by device/IP, and cap jumps per hour. Otherwise the line is gameable with throwaway tabs.
